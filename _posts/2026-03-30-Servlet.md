## 1. Servlet이란 무엇인가?

서블릿은 HTTP 요청을 처리하고 HTTP 응답을 생성하는 프로그램이다.  
서블릿은 단독으로 실행되지 않으며, 반드시 서블릿 컨테이너가 필요하다.


## 2. Servlet 동작 과정
서블릿의 흐름을 동작 과정을 통해 이해하고자 한다.

다음과 같이 서블릿을 등록했다고 가정하자.

```java
@WebServlet(name = "helloServlet", urlPatterns = "/hello")
```

사용자가 다음 URL에 접속하려고 한다. http://localhost:8080/hello?username=kim

### 2.1 브라우저에서 HTTP 요청

사용자가 URL을 입력하면 브라우저는 HTTP 요청을 생성한다.

```http
GET /hello?username=kim HTTP/1.1
Host: localhost:8080
```

이 요청에는 다음 정보가 포함되어 있다.

- 요청 방식: GET
- 요청 경로: /hello
- 쿼리 파라미터: username=kim

### 2.2 톰캣이 요청을 수신

톰캣은 웹 서버이자 서블릿 컨테이너로, 브라우저에서 보낸 HTTP 요청을 가장 먼저 받는다.

### 2.3 URL에 맞는 서블릿 찾기

톰캣은 요청 URL /hello를 확인하고, 다음 설정과 매칭되는 서블릿을 찾는다.

```java
@WebServlet(urlPatterns = "/hello")
```

### 2.4 request / response 객체 생성

톰캣은 요청을 처리하기 위해 두 객체를 생성한다.

HttpServletRequest → 요청 정보 <br>
HttpServletResponse → 응답 생성용

이 객체에는 HTTP 요청의 모든 정보가 담긴다.

예를 들어 
```java
request.getParameter("username"); // "kim"
```

### 2.5 서블릿 실행

톰캣은 해당 서블릿의 service() 메서드를 호출한다.
(* 맥북 단축키는 Ctrl + O)

```java
@Override
protected void service(HttpServletRequest request, HttpServletResponse response) {

    String username = request.getParameter("username");

    response.getWriter().write("hello " + username);
}
```

여기서 요청 데이터 추출, 비즈니스 로직 처리, 응답 생성과 같은 로직이 수행된다.

### 2.6 응답 생성

서블릿은 response 객체를 통해 HTTP 응답을 만든다.

```java
response.setContentType("text/plain");
response.setCharacterEncoding("UTF-8");
response.getWriter().write("hello kim");
```

### 2.7 브라우저로 응답 반환

톰캣은 서블릿이 만든 응답을 브라우저로 전달한다.

```http
HTTP/1.1 200 OK 
Content-Type: text/plain;charset=UTF-8
```
